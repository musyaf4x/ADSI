# ADSI Case Study — CORELASI Dashboard Admin

## Result

- **Baseline ADSI:** 34.5 / 100
- **Rounded:** 35 / 100
- **Band:** mild_moderate_slop
- **Verdict:** Visual foundation is strong, but the dashboard needs deeper product logic and operational affordances.
- **Target after repair:** below 20 / 100

## Score Breakdown

| Code | Area | Severity | Weighted Score | Key Finding |
|---|---|---:|---:|---|
| A | Realisme Data & Domain | 2.5 | 7.5 | Metric cards and table scope are not explicitly connected. |
| B | Kejelasan Workflow | 2 | 4.8 | Alert cards are not actionable enough. |
| C | Konsistensi Design System | 1 | 2.4 | Visual system is mostly consistent. |
| D | Hirarki Informasi | 1 | 2.0 | Hierarchy is strong; minor title repetition. |
| E | Copywriting & Lokalisasi | 2 | 4.0 | Some mixed/ambiguous language. |
| F | State & Status Logic | 2 | 4.0 | Status taxonomy is not fully clear. |
| G | Aksesibilitas | 1.5 | 3.0 | Pastel badges and dot-based cues need support. |
| H | Layout & Responsiveness | 1 | 1.6 | Desktop layout good; responsive behavior needs verification. |
| I | Orisinalitas Visual | 2 | 3.2 | Polished but somewhat generic admin-dashboard feel. |
| J | Production Readiness | 2 | 2.0 | Missing visible filters, drill-downs, and states. |

## Priority Repairs

1. Add data scope and data freshness copy.
2. Make problem cards clickable/actionable.
3. Standardize status taxonomy.
4. Improve Indonesian localization and microcopy clarity.
5. Add filter/drill-down controls and accessibility improvements.
